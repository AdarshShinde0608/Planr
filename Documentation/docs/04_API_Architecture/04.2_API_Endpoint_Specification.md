# 1. Authentication APIs

---

## Login

```
POST /auth/login
```

### Purpose

Authenticate user using Firebase ID Token.

### Authentication

❌ Not Required

### Request

```json
{
  "firebaseToken": "string"
}
```

### Response

```json
{
  "success": true,
  "data": {
    "user": {},
    "jwt": "token"
  }
}
```

---

## Register

```
POST /auth/register
```

### Request

```json
{
  "firebaseToken": "...",
  "email": "...",
  "fullName": "..."
}
```

### Response

```
201 Created
```

---

## Logout

```
POST /auth/logout
```

Invalidates the current session.

# API Security

Every protected endpoint requires:

```
Authorization: Bearer <Firebase Token>
```

Additional requirements:

* Firebase token verification
* User ownership validation
* Input sanitization
* Rate limiting
* Request logging

---

# API Versioning Policy

* All endpoints are under `/api/v1`.
* Breaking changes require a new version (`/api/v2`).
* Deprecated endpoints remain supported during migration.

---

# Antigravity Development Instructions

When implementing the REST API:

1. Follow the endpoint structure exactly as specified.
2. Validate all requests with Zod.
3. Authenticate protected routes using Firebase middleware.
4. Keep controllers thin and delegate logic to services.
5. Publish domain events after successful write operations.
6. Trigger AI agents through background jobs rather than synchronous calls where possible.
7. Return standardized success and error responses.
8. Implement pagination, filtering, sorting, and searching consistently.
9. Generate OpenAPI/Swagger documentation from the endpoint definitions.
10. Write integration tests for every endpoint before marking it complete.
