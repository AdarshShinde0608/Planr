# Purpose

Security is a core architectural concern for **The Last-Minute Life Saver**.

The platform manages:

* Personal schedules
* Goals
* Habits
* AI conversations
* Calendar events
* Voice interactions
* Productivity analytics

The security architecture ensures confidentiality, integrity, availability, and privacy across every layer of the system.

---

# Objectives

The security architecture should:

* Protect user data
* Secure AI interactions
* Prevent unauthorized access
* Protect APIs
* Secure infrastructure
* Encrypt sensitive information
* Detect attacks
* Support future compliance requirements

---

# Security Principles

The platform follows:

* Zero Trust
* Least Privilege
* Defense in Depth
* Secure by Default
* Privacy by Design
* Fail Securely

# JWT Strategy

Access Token

* Short lifetime (15 minutes)

Refresh Token

* Long lifetime (30 days)

Refresh tokens should be securely stored and rotated after use.

---

# Authorization

Use **Role-Based Access Control (RBAC)**.

Initial roles

| Role    | Permissions             |
| ------- | ----------------------- |
| User    | Personal resources only |
| Admin   | Platform administration |
| Support | Read-only support tools |

Every API must verify authorization before executing business logic.

---

# API Security

Protect every endpoint with:

* JWT validation
* Input validation
* Request size limits
* Rate limiting
* HTTPS only
* CORS policy

---

# Input Validation

Validate:

* JSON payloads
* File uploads
* Query parameters
* Path parameters
* AI prompts

Reject malformed requests immediately.

---

Default limits

| -------------- | ----------------- |

| AI Chat        | 60 requests/hour  |
| Task APIs      | 300 requests/hour |

# Prompt Injection Protection

Before every LLM request:

* Remove system prompt manipulation
* Ignore attempts to reveal hidden instructions
* Strip malicious instructions
* Limit accessible context
* Validate retrieved memories

---

# AI Output Validation

Validate responses for:

* Sensitive data leakage
* Malformed JSON
* Harmful output
* Invalid schedules
* Broken task references

---

# Encryption

## Data in Transit

* TLS 1.3
* HTTPS only

---

## Data at Rest

Encrypt:

* Database storage
* Backups
* Object storage
* Voice recordings
* AI memory

# Password Security

Requirements

* Never store plaintext passwords
* Use Firebase Authentication
* Enforce strong password policies
* Support password reset

---

# Secrets Management

Never commit secrets to source control.

Store securely:

* API keys
* Database credentials
* JWT secrets
* OAuth secrets
* Firebase credentials

Use environment variables or a cloud secret manager.

---

# File Upload Security

Validate:

* MIME type
* File extension
* Maximum size
* Virus scan (future)

Reject executable files.

# Best Practices

* Follow the principle of least privilege.
* Never trust client-side validation.
* Encrypt sensitive data both at rest and in transit.
* Rotate secrets regularly.
* Log security-relevant events.
* Review dependencies for vulnerabilities.
* Keep authentication and authorization separate.
* Validate every AI request and response.

---

# Antigravity Development Instructions

When implementing the Security Architecture:

1. Use Firebase Authentication with JWT access tokens and refresh token rotation.
2. Implement Role-Based Access Control (RBAC) using NestJS Guards.
3. Protect all APIs with authentication, authorization, validation, and rate limiting.
4. Introduce an AI Security Layer that validates prompts before LLM execution and validates responses before returning them to users.
5. Encrypt sensitive data at rest using AES-256 and enforce TLS 1.3 for all network communication.
6. Secure file uploads with MIME validation, size limits, and future malware scanning support.
7. Store all credentials in a secure secret management solution and never commit secrets to source control.
8. Generate comprehensive audit logs for authentication, authorization, administrative actions, and AI security events.
9. Implement security monitoring with automated alerts for suspicious behavior and repeated attack patterns.
10. Design the platform to support future compliance initiatives such as GDPR, SOC 2, and ISO 27001 without major architectural changes.
