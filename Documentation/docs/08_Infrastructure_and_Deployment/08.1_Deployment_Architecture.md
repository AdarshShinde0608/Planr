# Purpose

This document defines the deployment architecture for **The Last-Minute Life Saver**, ensuring the platform is secure, scalable, observable, and maintainable in production.

The deployment architecture supports:

* Web application
* Backend API
* AI Orchestrator
* AI Agents
* PostgreSQL
* Redis
* Background Workers
* Monitoring Stack
* CI/CD Pipeline

---

# Objectives

The deployment architecture should:

* Support Development, Staging, and Production environments
* Enable zero-downtime deployments
* Support horizontal scaling
* Isolate services
* Secure infrastructure
* Automate deployments
* Enable disaster recovery
* Monitor all services

---

# Environment Strategy

Maintain three independent environments.

| Environment | Purpose           |
| ----------- | ----------------- |
| Development | Local development |
| Staging     | QA & UAT          |
| Production  | Live users        |

Each environment has:

* Separate database
* Separate Redis
* Separate API keys
* Separate storage
* Separate monitoring

Never share production resources with lower environments.

# Best Practices

* Keep services stateless.
* Separate compute from storage.
* Use environment-specific secrets.
* Monitor everything.
* Automate deployments.
* Enable automatic backups.
* Use rolling deployments.
* Keep containers lightweight.
* Minimize downtime during releases.

---

# Antigravity Development Instructions

When implementing the deployment architecture:

1. Containerize every service using Docker.
2. Keep the Frontend, Backend API, AI Orchestrator, Workers, PostgreSQL, and Redis as separate services.
3. Build the local development environment with Docker Compose and ensure future compatibility with Kubernetes.
4. Configure GitHub Actions for automated testing, Docker image creation, and staged deployments.
5. Place Nginx (or another reverse proxy) in front of application services for TLS termination, routing, and security.
6. Implement health check endpoints for every service.
7. Integrate centralized monitoring (Prometheus, Grafana, Loki, Sentry).
8. Configure automated backups and disaster recovery procedures.
9. Manage all secrets through environment variables or a dedicated secret manager.
10. Design infrastructure to support horizontal scaling without application redesign.
