# Purpose

Observability provides complete visibility into the health, performance, reliability, and behavior of the entire platform.

Every request, AI decision, database query, background job, notification, calendar synchronization, and infrastructure event should be measurable and traceable.

The goal is to detect problems before users notice them and reduce the time required to diagnose and resolve incidents.

---

# Objectives

The Observability Platform should:

* Monitor every service
* Collect centralized logs
* Measure performance
* Detect failures automatically
* Monitor AI agents
* Track business KPIs
* Generate alerts
* Support incident investigation

# Structured Logging

Every log entry should contain

```json
{
  "timestamp": "",
  "level": "",
  "service": "",
  "requestId": "",
  "userId": "",
  "message": "",
  "duration": "",
  "metadata": {}
}
```

# Best Practices

* Use structured JSON logging.
* Never log passwords, tokens, or secrets.
* Assign a correlation ID to every request.
* Monitor AI usage independently from API metrics.
* Separate business metrics from infrastructure metrics.
* Create dashboards for different stakeholders.
* Review alerts regularly to reduce noise.
* Maintain runbooks for recurring incidents.

---

# Antigravity Development Instructions

When implementing the Monitoring, Logging & Observability platform:

1. Instrument every service using **OpenTelemetry** for distributed tracing.
2. Collect infrastructure and application metrics with **Prometheus**.
3. Store structured logs in **Loki** and visualize them through **Grafana**.
4. Configure **Sentry** for backend and frontend error tracking.
5. Implement correlation IDs across API requests, background jobs, AI agent executions, and external service calls.
6. Create dashboards for Engineering, AI Operations, Infrastructure, and Business KPIs.
7. Configure Alertmanager with severity-based notification rules.
8. Expose `/health`, `/ready`, and `/live` endpoints for every service.
9. Continuously monitor AI latency, token usage, cache efficiency, and cost.
10. Maintain runbooks and post-incident review processes to improve platform reliability.
