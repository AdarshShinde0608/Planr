# Product Requirements Document (PRD)

> Project: The Last-Minute Life Saver
> Version: 1.0
> Status: Planning
> Document Type: Product Requirements Document (PRD)

---

# Purpose

This document defines **what the product should do**, **who it serves**, **how users interact with it**, and the **functional and non-functional requirements**.

It serves as the single source of truth for designers, developers, AI coding agents, and testers.

---

# Product Vision

Build an AI-powered productivity companion that proactively helps users plan, prioritize, schedule, and complete tasks before deadlines are missed.

Instead of sending passive reminders, the application should actively guide users toward completing work.

---

# Product Goals

The application must:

- Reduce missed deadlines
- Reduce procrastination
- Generate realistic schedules
- Learn from user behavior
- Continuously improve recommendations
- Minimize manual planning

# Functional Requirements

## Authentication

Users can:

- Register
- Login
- Reset Password
- Update Profile

---

## Onboarding

The application collects:

- Wake-up time
- Sleep time
- Work hours
- College hours
- Gym schedule
- Meals
- Travel time
- Preferred focus hours
- Reminder preferences

---

## Task Management

Users can:

- Create tasks
- Edit tasks
- Delete tasks
- Mark tasks complete
- Add attachments
- Add recurring tasks

---

## AI Task Analysis

The Planner Agent must:

- Estimate duration
- Estimate difficulty
- Categorize tasks
- Detect deadlines
- Suggest subtasks

---

## AI Dynamic Scheduling

The Scheduler Agent must:

- Analyze fixed routine
- Read Google Calendar
- Find free slots
- Schedule tasks
- Add breaks
- Optimize workload
- Reschedule automatically

---

## Smart Reminders

The Reminder Agent must:

- Send contextual reminders
- Detect schedule risks
- Suggest departure times
- Escalate reminders if ignored

---

## Productivity Coaching

The Coach Agent must:

- Generate daily summaries
- Suggest focus sessions
- Recommend productivity improvements
- Produce weekly reports

---

## Goal Planning

The application must:

- Create long-term goals
- Break goals into milestones
- Generate weekly and daily plans

---

## Habit Tracking

Users can:

- Create habits
- Track streaks
- Monitor consistency

---

## Analytics

Display:

- Productivity score
- Completion rate
- Focus time
- Missed deadlines
- Weekly trends

# User Stories

### Authentication

As a new user,

I want to register securely,

so that my data is saved across devices.

---

### Scheduling

As a student,

I want AI to automatically organize my study sessions,

so I don't have to manually plan my day.

---

### Task Planning

As a professional,

I want AI to estimate task duration,

so I know when to start working.

---

### Reminders

As a busy user,

I want contextual reminders,

so I don't miss important commitments.

---

### Productivity

As a user,

I want to know where my time goes,

so I can improve my habits.

---

# Business Rules

- Fixed routines cannot be overwritten automatically.
- User-locked calendar events are immutable.
- AI recommendations must always be explainable.
- Tasks with earlier deadlines receive higher priority.
- AI may suggest but never force changes.

---

# Edge Cases

- Overlapping tasks
- Impossible schedules
- Missed deadlines
- Empty calendar
- No internet connection
- AI service unavailable
- Conflicting recurring events

---

# Acceptance Criteria

The MVP is complete when:

- Users can register and log in.
- Users can define their routine.
- Users can create tasks.
- AI analyzes tasks.
- AI generates schedules.
- Calendar sync works.
- Smart reminders are delivered.
- Productivity dashboard displays analytics.

---

# Success Metrics

- Increase task completion rate.
- Reduce missed deadlines.
- Improve schedule adherence.
- Increase daily productivity score.
- Reduce manual scheduling time.
