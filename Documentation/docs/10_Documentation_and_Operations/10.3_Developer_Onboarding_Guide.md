# Purpose

This guide helps new developers become productive as quickly as possible by providing a standardized onboarding process, development environment setup, repository overview, workflows, and engineering practices.

---

# Objectives

Every developer should be able to:

* Clone the project
* Configure the development environment
* Run the application locally
* Understand the architecture
* Execute tests
* Follow coding standards
* Make their first contribution
* Submit a Pull Request

Target onboarding time:

**Less than 2 hours**

# Install Dependencies

Preferred package manager:

```bash
pnpm install
```

Alternative:

```bash
npm install
```

---

# Environment Configuration

Create environment files:

```text
.env

.env.local
```

Minimum variables:

```text
DATABASE_URL=

REDIS_URL=

OPENAI_API_KEY=

JWT_SECRET=

SUPABASE_URL=

SUPABASE_KEY=
```

Never commit `.env` files.

---

# Start Infrastructure

Using Docker:

```bash
docker compose up -d
```

Starts:

* PostgreSQL
* Redis
* Supporting services

# Database Setup

Run:

```bash

```

Then:

```bash

```

Verify that sample data is available.

* Redis container
* REDIS_URL

---

## AI Requests Failing

Verify:

* API key
* Rate limits
* Internet connectivity
* Provider status

## Build Errors

Try:

```bash
rm -rf node_modules

pnpm install
```

Then rerun the build.

# Development Best Practices

* Keep Pull Requests focused.
* Write tests for new features.
* Update documentation alongside code.
* Reuse shared components.
* Review architecture before implementation.
* Prefer readability over cleverness.
* Ask for architectural review when introducing significant changes.

# Best Practices

* Read the project overview before coding.
* Review the architecture documentation.
* Keep feature branches short-lived.
* Run tests before every commit.
* Keep commits small and descriptive.
* Update documentation immediately after implementation.
* Seek code reviews for significant architectural changes.
* Follow established conventions rather than introducing new patterns.

---

# Antigravity Development Instructions

When onboarding new developers:

1. Provide access to the repository, required services, and documentation before the first development task.
2. Standardize local setup using Docker and environment templates.
3. Require all contributors to follow the project's coding standards, Git workflow, and documentation policies.
4. Automate setup steps where practical through scripts and package manager commands.
5. Ensure every developer can run the full application and test suite locally.
6. Encourage small initial contributions to familiarize new contributors with the codebase.
7. Maintain onboarding documentation as a living document that evolves with the project.
8. Review onboarding feedback periodically to reduce setup friction.
9. Keep developer tooling and dependency versions synchronized across the team.
10. Make onboarding reproducible so any developer can become productive with minimal manual assistance.
