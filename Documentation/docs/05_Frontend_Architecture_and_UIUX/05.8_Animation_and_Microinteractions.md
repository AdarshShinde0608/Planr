# 05.8 Animation & Microinteractions

## Overview

This document defines the animation and microinteraction system for **The Last-Minute Life Saver**.

Animations should enhance usability by providing feedback, improving navigation, and creating a polished user experience.

The motion system follows modern SaaS design principles:

- Purposeful
- Minimal
- Smooth
- Fast
- Consistent
- Accessible
- Performance Optimized

---

# Motion Design Principles

The animation system follows these principles:

### 1. Motion Has Purpose

Every animation must communicate:

- State change
- Navigation
- Feedback
- Progress
- Hierarchy

Avoid decorative animations that do not improve usability.

---

### 2. Fast Interactions

Animations should feel responsive.

Recommended durations:

| Type | Duration |
|-------|----------|
| Hover | 150ms |
| Click | 120ms |
| Page Transition | 250ms |
| Modal | 250ms |
| Drawer | 300ms |
| Toast | 250ms |

---

### 3. Natural Motion

Animations should use smooth easing.

Preferred easing:

```
ease-out
```

or

```
cubic-bezier(0.4, 0, 0.2, 1)
```

---

# Animation Categories

The application uses:

- Hover Animations
- Press Animations
- Navigation Animations
- Loading Animations
- Feedback Animations
- Theme Transition Animations
- AI Interaction Animations

---

# Hover Effects

Hover effects should be subtle.

Cards

```
TranslateY(-4px)
```

Scale

```
1.02
```

Shadow

Increase slightly.

Glow

Only on interactive elements.

---

# Button Animation

Hover

- Slight brightness increase
- Soft elevation
- Cursor pointer

Click

```
Scale(0.98)
```

Release

Return smoothly.

---

# Card Animation

Glass cards should respond to hover.

Hover

- Lift
- Shadow increase
- Border brightness increase

Do NOT rotate cards.

---

# Navigation Animation

Sidebar

Expand

```
300ms
```

Collapse

```
250ms
```

Menu items

Fade

Slide

Active indicator

Smooth movement.

---

# Navbar Animation

Navbar remains fixed.

When scrolling:

- Increase blur slightly
- Reduce transparency
- Add soft shadow

---

# Page Transition

Every page transition should include:

Opacity

```
0 → 1
```

Translate

```
Y: 8px → 0px
```

Duration

```
250ms
```

---

# Modal Animation

Open

- Fade
- Scale 0.96 → 1
- Blur background

Close

Reverse animation.

---

# Drawer Animation

Desktop

Slide from left.

Mobile

Slide from bottom.

---

# Toast Animation

Appear

```
Fade + Slide Up
```

Disappear

```
Fade + Slide Down
```

Duration

```
250ms
```

---

# Dropdown Animation

Animation

- Fade
- Scale
- Blur

Duration

```
180ms
```

---

# Tooltip Animation

Fade

Scale

```
0.96 → 1
```

No bouncing.

---

# Loading Animations

Preferred loading methods:

- Skeleton loaders
- Progress bars
- Circular loaders

Avoid spinning indicators for long operations whenever possible.

---

# Skeleton Loader

Characteristics

- Rounded corners
- Soft shimmer
- Theme-aware colors

Duration

```
1.5s infinite
```

---

# AI Thinking Animation

When AI generates content:

Display

- Animated pulse
- Typing indicator
- Gradient shimmer

Never block the UI.

---

# Progress Animation

Progress bars should animate smoothly.

Duration

```
400ms
```

---

# Notification Animation

Bell icon

When new notification arrives

- Gentle pulse
- Small badge animation

Avoid excessive shaking.

---

# Theme Transition

When switching themes:

Animate

- Background colors
- Glass opacity
- Border colors
- Shadows
- Text colors

Duration

```
250ms
```

Never flash white or black during transition.

---

# Search Animation

Global search (Ctrl + K)

Open

- Scale
- Fade
- Blur

Close

Reverse.

---

# Empty State Animation

Illustration

Small floating animation.

Opacity

Gentle breathing effect.

---

# Success Animation

Examples

- Task completed
- Goal achieved

Animation

- Checkmark draw
- Soft pulse
- Small confetti (optional)

---

# Error Animation

Use

- Light shake
- Red border transition

Avoid aggressive shaking.

---

# Scroll Animation

Reveal components using

- Fade
- TranslateY

Do NOT animate every element individually.

---

# Chart Animation

Charts should

- Animate on first render
- Count numbers smoothly
- Respect reduced motion settings

---

# Glass Hover Effect

Glass components should

Hover

- Increase blur slightly
- Increase border opacity
- Slight shadow enhancement

Do NOT increase transparency excessively.

---

# Accessibility

Respect

```
prefers-reduced-motion
```

If enabled:

- Disable non-essential animations
- Keep essential feedback only
- Remove parallax
- Remove floating effects

---

# Performance Guidelines

Animations must:

- Use transform
- Use opacity
- Avoid layout reflow
- Avoid animating width or height
- Avoid excessive blur updates

Prefer GPU-accelerated properties.

---

# Framer Motion Guidelines

Recommended for:

- Page transitions
- Modal animations
- Drawer animations
- Shared layout transitions
- Gesture interactions

Avoid using Framer Motion for simple hover states that CSS can handle efficiently.

- Improves usability
- Feels natural
- Fast
- Theme aware
- Accessible
- Consistent
- GPU accelerated
- Doesn't distract the user

# Summary

The animation system provides a premium user experience through subtle, meaningful motion that enhances clarity rather than distracting from it.

By combining fast interactions, glassmorphism, accessibility, and performance optimization, the interface feels modern, polished, and responsive across desktop, tablet, and mobile devices.
