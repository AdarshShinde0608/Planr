# Purpose

This document defines the frontend architecture for **The Last-Minute Life Saver**.

The frontend is responsible for:

* Delivering a fast and responsive user experience
* Displaying AI-generated schedules
* Managing local UI state
* Synchronizing with backend services
* Supporting real-time updates
* Providing an accessible and scalable interface

The architecture follows modern React and Next.js best practices.

# Development Standards

* TypeScript strict mode enabled.
* Functional components only.
* No inline business logic in pages.
* Reusable components before duplication.
* Shared types imported from the API contract.
* Feature isolation wherever possible.

---

# Best Practices

* Keep pages thin.
* Prefer composition over inheritance.
* Separate UI from business logic.
* Avoid unnecessary global state.
* Keep components focused on a single responsibility.
* Use memoization only when measurable benefits exist.
* Prefer declarative patterns.

---

# Antigravity Development Instructions

When implementing the frontend:

1. Initialize a Next.js App Router project with TypeScript.
2. Configure Tailwind CSS, shadcn/ui, and Framer Motion.
3. Create a feature-based folder structure.
4. Implement centralized API services using Axios.
5. Use TanStack Query for server state.
6. Use Zustand only for global UI state.
7. Keep business logic inside custom hooks.
8. Implement layouts before individual pages.
9. Ensure every page supports loading, empty, and error states.
10. Maintain strict TypeScript typing using the shared API schemas.
