# 05.5 UI Component Architecture

## Overview

This document defines the reusable UI component architecture for **The Last-Minute Life Saver**.

The objective is to create a consistent, scalable, accessible, and theme-aware component library built on modern design principles.

The component library follows:

- Minimalistic Design
- Glassmorphism
- Light & Dark Theme Support
- Mobile First Design
- Accessibility First
- Reusability
- Performance

---

# Design Philosophy

Every component should follow these principles:

- Minimal visual noise
- Maximum readability
- Consistent spacing
- Smooth animations
- Premium appearance
- Soft glass surfaces
- Rounded geometry
- Theme awareness
- Accessibility

Every component must look good in both:

- 🌙 Dark Theme
- ☀️ Light Theme

without requiring separate implementations.

# Component Design Rules

Every component must use:

- Theme Tokens
- CSS Variables
- Glass Surfaces
- Responsive Layout
- Semantic Colors

Never use:

- Hardcoded colors
- Hardcoded spacing
- Inline styles

---

# Glassmorphism Rules

Glass components should have:

Background

```
background: var(--glass-bg);
```

Border

```
border: 1px solid var(--glass-border);
```

Blur

```
backdrop-filter: blur(var(--glass-blur));
```

Shadow

```
box-shadow: var(--glass-shadow);
```

---

# Animation Rules

Hover

- Slight lift
- Soft glow
- Scale 1.02

Click

- Scale 0.98

- Fade
- Slide
- Blur

```

```

# Theme Behaviour

Every component automatically adapts using

```
data-theme="light"

data-theme="dark"
```

No component should manually switch colors.

# Accessibility

Every component must support:

- Keyboard navigation
- Focus indicators
- Screen readers
- WCAG AA contrast

# Performance Guidelines

Components should:

- Avoid unnecessary re-rendering
- Support lazy loading
- Use memoization where appropriate
- Avoid excessive blur stacking
