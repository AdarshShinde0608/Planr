# 05.2 Design System (Theme-Aware + Glassmorphism)

## Overview

The Design System defines the **visual language** of The Last-Minute Life Saver.

It is fully integrated with:

- 05.10 Theme System (Light/Dark)
- Glassmorphism UI engine
- Minimalistic SaaS design principles

---

## Core Philosophy

The UI is built on 4 principles:

### 1. Minimalism
- Remove visual noise
- Prioritize content
- Reduce unnecessary borders

### 2. Glassmorphism
- Transparent surfaces
- Soft blur layers
- Floating UI elements

### 3. Theme Adaptability
- Every element responds to theme changes
- No hardcoded colors allowed

### 4. Consistency
- Unified spacing system
- Shared tokens across all components

---

## Glassmorphism System
```css
Standard Glass Card
background: var(--glass-bg);
backdrop-filter: blur(var(--glass-blur));
border: 1px solid var(--glass-border);
border-radius: var(--radius-lg);
```

## Light Theme Glass Behavior
Reduced blur
Increased opacity
Soft shadow instead of glow

## Dark Theme Glass Behavior
Strong blur
Lower opacity
Subtle neon edge glow

## Shadow System
sm = soft elevation
md = card elevation
lg = modal elevation
xl = floating UI elevation

Dark mode uses glow-based shadows.

Light mode uses depth-based shadows.

## Z-Index System
dropdown  = 100
sticky    = 200
modal     = 500
toast     = 700
tooltip   = 900


## Component Styling Rules
Every UI component must:

Use design tokens only
Support both themes
Support glass mode
Avoid hardcoded values
Use consistent spacing system

## Example Component Rule
❌ Incorrect
background: white;
padding: 10px;
border-radius: 5px;

✅ Correct
background: var(--glass-bg);
padding: var(--space-md);
border-radius: var(--radius-md);
