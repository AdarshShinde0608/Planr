# Purpose

Performance testing ensures the platform remains responsive, reliable, and scalable as user traffic, AI requests, and background workloads increase.

Unlike functional testing, performance testing focuses on **speed**, **throughput**, **resource utilization**, and **system stability**.

---

# Objectives

The performance testing strategy should verify:

* API responsiveness
* AI latency
* Database scalability
* Queue throughput
* Multi-agent concurrency
* Infrastructure utilization
* Auto-scaling behavior
* Production readiness

# Best Practices

* Use production-like datasets.
* Test realistic user behavior.
* Monitor every infrastructure component.
* Benchmark AI agents independently and together.
* Execute soak tests before major releases.
* Continuously compare results with previous baselines.
* Treat performance regressions as defects.
* Record benchmark history for trend analysis.

---

# Antigravity Development Instructions

When implementing Performance, Load & Stress Testing:

1. Build automated performance test suites using **k6** for APIs and backend services.
2. Benchmark every AI Agent independently and as part of complete multi-agent workflows.
3. Measure latency, throughput, error rates, CPU, memory, database performance, Redis performance, and queue utilization.
4. Execute load, stress, spike, soak, scalability, and capacity tests before production releases.
5. Validate auto-scaling behavior using production-like infrastructure.
6. Define and enforce performance budgets for APIs, frontend assets, AI responses, and scheduler execution.
7. Integrate performance benchmarking into CI/CD pipelines and maintain historical performance baselines.
8. Generate detailed benchmark reports with infrastructure metrics and AI cost analysis.
9. Continuously monitor production performance using Prometheus and Grafana dashboards.
10. Block releases if critical performance targets or scalability requirements are not satisfied.
