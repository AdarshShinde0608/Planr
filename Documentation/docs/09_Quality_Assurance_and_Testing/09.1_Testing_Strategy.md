# Purpose

This document defines the complete Quality Assurance strategy for **The Last-Minute Life Saver**.

The application consists of:

* Frontend
* Backend
* Database
* AI Orchestrator
* Planner Agent
* Scheduler Agent
* Coach Agent
* Reminder Agent
* Background Workers
* External Integrations

Every component must be validated before deployment.

Testing is treated as a first-class engineering discipline rather than a final development phase.

---

# Objectives

The testing strategy should ensure:

* Functional correctness
* High reliability
* Production stability
* AI consistency
* Secure releases
* Fast regression detection
* Automated quality gates

---

# Testing Philosophy

# Exit Criteria

A release is approved only when:

* All tests pass.
* Coverage thresholds are met.
* No critical or high-severity defects remain.
* Performance targets are satisfied.
* Security scans pass.

# Best Practices

* Write tests alongside implementation.
* Keep tests deterministic.
* Mock external dependencies.
* Test edge cases and failure paths.
* Automate regression testing.
* Review flaky tests immediately.
* Measure coverage but prioritize meaningful tests over percentage alone.
* Continuously refine AI evaluation datasets as the application evolves.

---

# Antigravity Development Instructions

When implementing the Testing Strategy:

1. Adopt the Testing Pyramid with approximately 70% unit tests, 20% integration tests, and 10% end-to-end tests.
2. Require automated testing for every new feature before merge approval.
3. Create dedicated test suites for each AI Agent and for complete multi-agent workflows.
4. Build deterministic AI tests using mocked LLM responses where appropriate.
5. Integrate all tests into the GitHub Actions CI/CD pipeline.
6. Enforce coverage thresholds before deployment.
7. Include performance, load, security, and accessibility testing in release validation.
8. Maintain isolated test environments and datasets.
9. Continuously execute regression tests on every release candidate.
10. Treat failing automated tests as release blockers until resolved.
