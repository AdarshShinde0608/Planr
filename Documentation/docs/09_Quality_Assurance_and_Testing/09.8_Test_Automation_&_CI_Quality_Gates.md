# Purpose

This document defines the automated quality assurance pipeline for **The Last-Minute Life Saver**.

Every code change must automatically pass predefined quality gates before it can be merged or deployed.

Automation ensures:

* Consistent quality
* Fast feedback
* Reliable deployments
* Reduced human error
* Continuous verification

---

# Objectives

The automation pipeline should:

* Execute tests automatically
* Prevent broken builds
* Enforce coding standards
* Validate security
* Measure performance
* Verify AI quality
* Protect production

---

# Continuous Quality Philosophy

Every code change follows the same validation process.

```text
Developer

↓

Commit

↓

Pull Request

↓

Automated Validation

↓

Code Review

↓

Merge

↓

Deployment
```

Quality is continuously enforced—not inspected at the end.

# Best Practices

* Fail fast on critical issues.
* Automate repetitive validation.
* Keep CI pipelines deterministic.
* Execute tests in parallel where possible.
* Monitor pipeline duration.
* Treat quality gates as non-optional.
* Archive reports for compliance and auditing.
* Continuously improve pipeline performance.

---

# Antigravity Development Instructions

When implementing Test Automation & CI Quality Gates:

1. Configure GitHub Actions as the primary CI/CD automation platform.
2. Execute linting, formatting, type checking, unit tests, integration tests, AI evaluations, security scans, and build verification on every Pull Request.
3. Enforce mandatory branch protection rules for all production branches.
4. Block merges and deployments if any required quality gate fails.
5. Run nightly pipelines containing complete regression, performance, AI benchmark, and security suites.
6. Generate and archive artifacts including coverage reports, security reports, AI evaluation reports, and deployment metadata.
7. Detect and report flaky tests automatically.
8. Execute test suites in parallel to minimize pipeline duration.
9. Publish quality dashboards showing coverage, AI quality, security posture, and deployment health.
10. Continuously optimize CI execution time while maintaining complete validation coverage.
