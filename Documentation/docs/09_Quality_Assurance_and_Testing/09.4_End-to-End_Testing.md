# Purpose

End-to-End (E2E) testing validates the complete application from the user's perspective.

Unlike Unit and Integration testing, E2E tests verify that all components—including the frontend, backend, AI agents, database, background workers, and external services—work together to deliver a seamless user experience.

---

# Objectives

E2E testing should verify:

* Complete user journeys
* Cross-service workflows
* Multi-agent AI collaboration
* UI correctness
* External integrations
* Production readiness
* Regression protection

---

# E2E Testing Philosophy

Every critical feature should be testable exactly as a user would use it.

The application should be validated through realistic scenarios rather than isolated component behavior.

# Best Practices

* Test only real user behavior.
* Avoid implementation-specific assertions.
* Keep tests independent.
* Reset test data before each run.
* Record traces for failed executions.
* Execute smoke tests after every deployment.
* Prioritize stability over quantity of E2E tests.
* Continuously update scenarios as new features are introduced.

---

# Antigravity Development Instructions

When implementing End-to-End testing:

1. Use **Playwright** as the primary E2E automation framework.
2. Build automated tests for every critical user journey from registration through daily productivity workflows.
3. Validate complete multi-agent AI execution, including Planner, Scheduler, Coach, and Reminder Agents.
4. Include browser compatibility, responsive layout, accessibility, and visual regression checks.
5. Record screenshots, videos, traces, and logs for every failed execution.
6. Execute smoke tests automatically after each production deployment.
7. Maintain isolated E2E environments using dedicated test accounts and datasets.
8. Ensure all E2E tests remain deterministic by mocking only non-essential third-party variability where appropriate.
9. Integrate the E2E suite into the CI/CD pipeline with mandatory pass requirements before release.
10. Continuously expand E2E scenarios as new features and AI capabilities are added.
