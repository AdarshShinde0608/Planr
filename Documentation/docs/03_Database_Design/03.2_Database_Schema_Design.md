# Validation Rules

Every schema should validate:

* Required fields
* Maximum string lengths
* Date formats
* Enum values
* Object references
* Duplicate prevention
* Soft delete status

Use Zod for request validation and Mongoose schema validation for persistence.

* Use document versioning.

# Antigravity Development Instructions

When implementing database schemas:

1. Create a separate Mongoose model for each collection.
2. Implement a reusable Base Schema for shared audit fields.
3. Add all required indexes during schema definition.
4. Keep AI-generated metadata isolated from user-entered data.
5. Use references for relationships between collections.
6. Enable timestamps and optimistic versioning.
7. Implement soft deletes for business entities.
8. Keep schemas backward-compatible to support future migrations.
9. Validate all persisted data using Mongoose and all incoming requests using Zod.
10. Ensure every schema is documented with field descriptions and ownership information.
