# Scalability Considerations

The architecture is designed to support:

* Millions of users
* Millions of tasks
* Multiple worker instances
* Horizontal MongoDB scaling
* Read replicas
* Distributed AI workers
* Future collaborative workspaces

No major architectural changes should be required to scale.

---

# Best Practices

* Maintain single ownership for each collection.
* Communicate through events rather than direct service calls.
* Keep AI context retrieval minimal.
* Use background workers for long-running operations.
* Avoid circular dependencies between collections.
* Store AI-generated metadata separately from user input.
* Update analytics asynchronously.

---

# Antigravity Development Instructions

When implementing the data layer:

1. Preserve the collection relationships defined in the ERD.
2. Ensure every collection references the owning user through `userId`.
3. Route all long-running workflows through the Event Bus.
4. Restrict each service and AI agent to its designated write responsibilities.
5. Keep AI agents independent and event-driven.
6. Optimize context retrieval by loading only required collections.
7. Ensure background workers update dependent collections asynchronously.
8. Maintain audit trails for all major state changes.
9. Avoid direct cross-service database writes.
10. Validate all relationships through repositories before persistence.
