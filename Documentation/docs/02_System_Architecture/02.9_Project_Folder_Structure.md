# Purpose

This document defines the complete project structure for **The Last-Minute Life Saver**.

The repository follows a **monorepo architecture** to keep frontend, backend, shared types, AI prompts, and documentation in one place while maintaining clear boundaries.

Goals:

* Easy navigation
* Feature-based organization
* High maintainability
* AI-friendly structure
* Scalable architecture
* Clean separation of concerns

---

# Naming Conventions

## Files

Use kebab-case.

Examples:

```text
task.service.ts

planner.agent.ts

notification.worker.ts
```

---

## Classes

Use PascalCase.

Examples:

```text
TaskService

PlannerAgent

ReminderWorker
```

---

## Functions

Use camelCase.

Examples:

```text
createTask()

generateSchedule()

sendReminder()
```

---

## Constants

Use UPPER_SNAKE_CASE.

Examples:

```text
DEFAULT_BREAK_DURATION

MAX_RETRY_ATTEMPTS
```

# Import Rules

Preferred import order:

1. External libraries
2. Shared packages
3. Internal modules
4. Relative imports

Avoid deep relative imports such as:

```text
../../../../utils
```

Use path aliases instead.

Example:

```typescript
import { Task } from "@shared/types";
import { PlannerAgent } from "@ai-core/agents";
```

# Code Ownership Principles

* UI components belong in `packages/ui`.
* Shared business logic belongs in `packages/shared`.
* AI logic belongs in `packages/ai-core`.
* Prompts belong in `packages/ai-prompts`.
* Application-specific code remains inside `apps/web` or `apps/api`.

Avoid duplicating logic across applications.

---

# Best Practices

* Organize by feature before type where appropriate.
* Keep files focused on a single responsibility.
* Avoid circular dependencies.
* Store prompts outside source code.
* Reuse shared types and utilities.
* Use path aliases for imports.
* Document public modules.
* Keep configuration centralized.

---

# Antigravity Development Instructions

When creating the repository:

1. Generate the monorepo exactly as defined.
2. Configure pnpm workspaces and TurboRepo.
3. Create all top-level directories before writing code.
4. Use shared packages for reusable logic.
5. Separate AI logic into `packages/ai-core`.
6. Store prompts in `packages/ai-prompts`.
7. Configure path aliases across all projects.
8. Enforce naming conventions through linting.
9. Keep documentation synchronized with implementation.
10. Do not introduce feature code outside the defined module boundaries.
