# Purpose

This document defines the frontend architecture of **The Last-Minute Life Saver**.

It specifies:

* Project structure
* Folder organization
* Routing
* Component architecture
* State management
* API communication
* Authentication flow
* UI standards
* Theme system
* Performance strategy

The frontend should be built as a scalable and maintainable application that communicates only with the backend API.

# State Management

Use **Zustand** for global state.

Stores

* User Store
* Authentication Store
* Task Store
* Schedule Store
* Goal Store
* Habit Store
* Notification Store
* AI Assistant Store
* Theme Store

Avoid unnecessary global state. Server data should be managed using **TanStack Query**.

---

# API Communication

The frontend must **never** communicate directly with AI providers or external services.

All requests follow:

```text
Frontend

↓

Backend REST API

↓

AI Orchestrator

↓

Database
```

---

# Authentication Flow

```text
User Login

↓

Firebase Authentication

↓

JWT Token

↓

Backend Verification

↓

User Profile

↓

Dashboard
```

Protected routes should redirect unauthenticated users to the login page.

# Error Handling

Every page should gracefully handle:

* Network errors
* API failures
* Empty data
* Unauthorized access

Provide actionable messages and retry options.

---

# Accessibility

The application should:

* Support keyboard navigation
* Use semantic HTML
* Maintain sufficient color contrast
* Provide ARIA labels where required

---

# Performance Guidelines

* Lazy-load large components
* Optimize images
* Cache API responses
* Minimize unnecessary re-renders
* Use code splitting
* Debounce search inputs

---

# Development Standards

* Use TypeScript everywhere.
* Prefer functional components.
* Keep components small and reusable.
* Separate UI from business logic.
* Avoid duplicated code.
* Follow consistent naming conventions.

---

# Antigravity Development Instructions

When implementing the frontend:

1. Generate the folder structure exactly as defined.
2. Build shared UI components first.
3. Implement layouts before feature pages.
4. Connect pages only to backend APIs.
5. Do not embed AI logic in the frontend.
6. Keep state management modular.
7. Follow responsive and accessibility guidelines.
