# Purpose

This document defines the complete Artificial Intelligence architecture of **The Last-Minute Life Saver**.

The application is built around a **Multi-Agent AI Architecture**, where each AI agent has a specialized responsibility.

Instead of relying on one large prompt to perform every task, the system decomposes complex decisions into multiple expert agents coordinated by an **AI Orchestrator**.

This design improves:

* Accuracy
* Scalability
* Maintainability
* Explainability
* Performance

---

# AI Philosophy

The AI should not replace the user.

Instead, it should:

* Think with the user
* Plan for the user
* Recommend actions
* Learn continuously
* Adapt schedules
* Prevent deadline failures

The AI acts as a **Productivity Operating System** rather than a chatbot.

---

# AI Principles

Every AI decision must be:

* Explainable
* Context-aware
* Personalized
* Non-destructive
* Reversible
* Privacy-respecting
* Human-controlled

The AI may recommend actions but must never silently modify critical user data.

---

# AI Orchestrator

The AI Orchestrator is the brain of the backend AI layer.

It is responsible for:

* Selecting which agents should execute
* Passing context between agents
* Managing execution order
* Combining outputs
* Handling retries and fallbacks
* Caching AI responses

The frontend **never communicates directly** with any AI agent.

---

# AI Context Engine

Before invoking any AI agent, the orchestrator builds a shared context.

Context includes:

* User profile
* Daily routine
* Calendar events
* Existing schedule
* Current tasks
* Goals
* Habits
* Productivity statistics
* Historical behavior
* User preferences
* Current date and time

This shared context is passed to each relevant agent.

---

# Planner Agent

## Purpose

Understand every task added by the user.

## Inputs

* Task title
* Description
* Deadline
* Attachments
* User profile
* Existing workload

## Responsibilities

* Estimate duration
* Predict difficulty
* Identify category
* Detect dependencies
* Generate subtasks
* Suggest optimal start date
* Estimate risk of missing deadline

## Outputs

```json
{
  "estimatedDuration": "4 hours",
  "difficulty": "Medium",
  "priority": "High",
  "category": "College",
  "recommendedStartDate": "Today",
  "deadlineRisk": "Moderate",
  "subtasks": []
}
```

Planner Agent **never modifies schedules**.

---

# Scheduler Agent

## Purpose

Generate realistic schedules.

## Inputs

* Planner output
* User routine
* Calendar
* Existing schedule
* Goals
* Habits
* Deadlines

## Responsibilities

* Find free time
* Allocate tasks
* Insert breaks
* Prevent conflicts
* Split long tasks
* Balance workload
* Regenerate schedules

## Rules

Never overwrite:

* Locked events
* Sleep schedule
* Fixed work hours
* Fixed classes

---

# Reminder Agent

## Purpose

Ensure users never miss commitments.

## Responsibilities

* Context-aware reminders
* Travel reminders
* Deadline reminders
* Escalating notifications
* Smart nudges
* Follow-up reminders

Examples:

* "Leave in 15 minutes for your meeting."
* "You have a free hour—this is a good time to finish your assignment."

---

# Coach Agent

## Purpose

Improve long-term productivity.

## Responsibilities

* Analyze work habits
* Generate daily summaries
* Recommend focus sessions
* Suggest habit improvements
* Weekly productivity reports
* Monthly reflections

The Coach Agent does **not** schedule tasks or modify deadlines.

# Agent Communication

Agents communicate **only** through the AI Orchestrator.

```text
Task Created

↓

Planner

↓

Scheduler

↓

Reminder

↓

Coach

↓

Return
```

Agents must never call one another directly.

---

# AI Memory

The system maintains three memory layers.

## Short-Term Memory

Stores:

* Current conversation
* Current task
* Today's schedule

Expires frequently.

---

## Medium-Term Memory

Stores:

* Weekly productivity
* Recent habits
* Recent schedules

Used for recommendations.

---

## Long-Term Memory

Stores:

* Preferred focus hours
* Average task completion time
* Habit history
* User preferences

Used to personalize AI behavior.

# Prompt Design

Each agent receives:

* System Prompt
* Context
* User Data
* Current Task
* Output Schema

Responses must be structured JSON whenever possible.

---

# AI Safety Rules

The AI must never:

* Delete tasks automatically
* Modify deadlines without confirmation
* Cancel calendar events
* Access unauthorized data
* Share personal information
* Invent missing facts

If confidence is low, the AI should request clarification rather than guessing.

---

# Explainability

Every AI recommendation should include a reason.

Example:

> "This task is scheduled for 7 PM because it requires high focus and your productivity is typically highest between 6 PM and 9 PM."

---

# Failure Handling

If an AI agent fails:

1. Retry once.
2. Use cached results if available.
3. Fall back to rule-based logic.
4. Notify the backend.
5. Continue application flow without blocking the user.

The application must remain usable even when AI services are unavailable.

---

# Performance Strategy

* Cache repeated analyses.
* Batch AI requests where possible.
* Limit token usage.
* Reuse shared context.
* Execute independent agents asynchronously when appropriate.

---

# Future AI Agents

The architecture should support adding new agents without modifying existing ones.

Potential additions:

* Meeting Preparation Agent
* Email Assistant Agent
* Health & Wellness Agent
* Finance Planning Agent
* Team Collaboration Agent
* Study Planner Agent
* Interview Preparation Agent

---

# AI Development Guidelines

* One responsibility per agent.
* Structured JSON outputs.
* No agent-to-agent direct communication.
* Prompts stored externally.
* Version prompts.
* Log AI decisions.
* Monitor confidence scores.
* Keep prompts deterministic where possible.

---

# Antigravity Development Instructions

When implementing the AI layer:

1. Create the AI Orchestrator before implementing individual agents.
2. Each agent should be an independent service with a clear interface.
3. Use typed request and response models.
4. Store prompts in external files, not inline code.
5. Validate AI outputs against predefined schemas.
6. Log execution time and failures.
7. Build fallback logic for every AI operation.
8. Design the system to support adding future agents without changing existing code.
