# Purpose

This document defines the **overall system architecture** of **The Last-Minute Life Saver**.

It provides a complete high-level view of how all components interact, including:

* Frontend
* Backend
* AI Multi-Agent System
* Database
* Background Workers
* External Integrations
* Notification Services

This document acts as the **master architecture reference** for all future development.

---

# Architecture Philosophy

The application follows an **AI-First, Service-Oriented Architecture**.

Instead of treating AI as an optional feature, AI is the central decision-making engine.

The backend coordinates business logic while specialized AI agents perform reasoning and planning.

---

# Design Principles

The architecture follows these principles:

* AI First
* Modular Design
* Service-Oriented
* Event Driven
* Scalable
* Explainable AI
* Separation of Concerns

* API First Development

# Core Components

## Frontend Layer

Responsibilities

* Authentication
* Dashboard
* Task Management
* Calendar
* Analytics
* Settings
* AI Chat
* Voice Interface

The frontend **never communicates directly with AI services**.

All requests must pass through the backend.

---

## Backend Layer

Acts as the application's central controller.

Responsibilities

* Authentication
* Validation
* Authorization
* Database operations
* Business rules
* AI orchestration
* Event publishing
* Notification scheduling

No AI business logic should exist inside controllers.

Controllers only coordinate requests.

---

## AI Orchestrator

The AI Orchestrator coordinates communication between all AI agents.

It decides:

* Which agent should execute
* Execution order
* Context sharing
* Result aggregation
* Error handling

Example

```
User creates task

↓

Planner Agent

↓

Scheduler Agent

↓

Reminder Agent

↓

Coach Agent

↓

Return response
```

The frontend never communicates directly with individual AI agents.

# Multi-Agent AI Layer

The application uses four independent AI agents.

---

## Planner Agent

Purpose

Understand every task.

Responsibilities

* Estimate duration
* Detect difficulty
* Categorize task
* Generate subtasks
* Detect dependencies

Input

Task

Output

Structured task information

---

## Scheduler Agent

Purpose

Generate realistic schedules.

Responsibilities

* Read user routine
* Read calendar
* Find free slots
* Allocate work sessions
* Add breaks
* Optimize workload
* Reschedule missed tasks

---

## Reminder Agent

Purpose

Prevent missed commitments.

Responsibilities

* Smart reminders
* Deadline reminders
* Context-aware reminders
* Escalation
* Notification timing

---

## Coach Agent

Purpose

Improve long-term productivity.

Responsibilities

* Daily reflection
* Weekly reports
* Productivity analysis
* Motivation
* Habit recommendations

# Error Handling Strategy

If Planner Agent fails

Return task without AI analysis.

Retry in background.

---

If Scheduler fails

Keep existing schedule.

Notify user.

---

If Reminder fails

Retry automatically.

---

If OpenAI API unavailable

Application continues using cached information and basic scheduling rules.

# Architecture Decisions

| Decision                    | Reason                                 |
| --------------------------- | -------------------------------------- |
| React + Next.js             | Modern, scalable frontend              |
| Node.js + Express           | Fast API development                   |
| MongoDB                     | Flexible document storage              |
| Redis                       | Caching and job queues                 |
| Multi-Agent AI              | Separation of AI responsibilities      |
| Event-Driven Architecture   | Loose coupling and scalability         |
| Google Calendar Integration | Existing user schedule synchronization |
| Firebase Cloud Messaging    | Cross-platform notifications           |
