# Purpose

Google Calendar is one of the primary data sources used by the Scheduler Agent.

The integration enables AI to:

* Import existing calendar events
* Detect free time
* Detect scheduling conflicts
* Publish AI-generated schedules
* Keep both systems synchronized
* Adapt schedules when the calendar changes

The Scheduler Agent should never create schedules without considering calendar availability.

---

# Objectives

The Calendar Integration should:

* Support secure OAuth authentication
* Import calendar events
* Export AI schedules
* Maintain two-way synchronization
* Detect scheduling conflicts
* Analyze free/busy time
* Support multiple calendars
* Recover automatically from synchronization failures

# Incremental Synchronization

Avoid downloading the entire calendar repeatedly.

Pipeline:

```

↓

↓

Update Local Database
```

This reduces API usage and improves performance.

# Best Practices

* Use incremental synchronization whenever possible.
* Never overwrite user-created events.
* Keep AI-generated events clearly identifiable.
* Respect user approval settings.
* Store timestamps in UTC.
* Monitor synchronization health continuously.
* Handle webhook failures gracefully.
* Support multiple calendars.

---

# Antigravity Development Instructions

When implementing Google Calendar Integration:

1. Build a dedicated **Calendar Integration Service** between Google Calendar and the AI Orchestrator.
2. Authenticate users using OAuth 2.0 and securely store access and refresh tokens.
3. Support one-way import, one-way export, and full two-way synchronization.
4. Import calendar events and expose free/busy information to the Scheduler Agent.
5. Publish AI-generated schedules as calendar events after user approval (default behavior).
6. Implement webhook listeners to detect calendar changes in near real time.
7. Use incremental synchronization tokens to reduce API usage.
8. Handle recurring events, timezone conversions, and offline synchronization correctly.
9. Protect all calendar data through encryption and secure token management.
10. Continuously monitor synchronization performance, conflicts, and API quota usage.
