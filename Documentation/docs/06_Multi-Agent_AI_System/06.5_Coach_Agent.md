# Purpose

The **Coach Agent** acts as the user's personal productivity coach.

Unlike the Planner Agent and Scheduler Agent, it does **not** create tasks or schedules.

Instead, it continuously analyzes user behavior to:

* Improve productivity
* Encourage consistency
* Prevent burnout
* Reinforce habits
* Provide personalized coaching
* Generate insights
* Recommend better working strategies

Its mission is to help users become more productive over time—not just complete today's tasks.

---

The Coach Agent is responsible for:

* Productivity analysis
* Focus coaching

# Motivation Engine

The Coach Agent generates short motivational messages.

Examples:

* "You're one task away from today's goal."
* "You've improved your focus time by 18% this month."
* "Excellent consistency—keep the streak alive."

Messages should be positive, factual, and personalized.

---

# Insight Generation

Examples:

* You consistently finish coding tasks faster than estimated.
* Morning focus sessions have a 94% completion rate.
* Gym sessions improve evening productivity.
* You are most productive on Tuesdays.

Insights must be supported by data.

---

# Recommendation Engine

Recommendations should be:

* Actionable
* Personalized
* Evidence-based
* Time-sensitive

Example:

```text id="coach010"

Move difficult tasks to your morning focus block.

```

# Reflection Generation

At the end of each day:

Questions:

* What went well?
* What was difficult?
* What should improve tomorrow?

The agent summarizes the day's performance and prepares suggestions for the next day.

# Best Practices

* Base every recommendation on measurable data.
* Encourage consistency rather than perfection.
* Avoid repetitive motivational messages.
* Keep advice concise and actionable.
* Prioritize user well-being over maximum productivity.
* Explain why each recommendation is given.
* Continuously adapt to evolving user behavior.

---

# Antigravity Development Instructions

When implementing the Coach Agent:

1. Build the Coach Agent as an independent AI service coordinated by the AI Orchestrator.
2. Consume historical analytics, schedules, goals, and habits instead of raw user prompts.
3. Calculate productivity scores using configurable weighted metrics.
4. Implement burnout detection based on workload, break patterns, and historical trends.
5. Generate evidence-based recommendations with explanations.
6. Produce weekly and monthly reports automatically through scheduled background jobs.
7. Return responses using the defined JSON schema.
8. Store coaching history to evaluate recommendation effectiveness over time.
9. Avoid modifying schedules or tasks directly; only provide guidance.
10. Continuously improve recommendation quality using user feedback and acceptance metrics.
