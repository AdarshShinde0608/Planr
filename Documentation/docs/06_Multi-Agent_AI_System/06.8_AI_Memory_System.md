# Purpose

The AI Memory System enables all AI agents to learn from previous user interactions.

Unlike traditional productivity applications that forget previous behavior after every interaction, this system continuously builds a personalized understanding of the user.

The memory system allows the application to answer questions such as:

* When is the user most productive?
* Which reminders are most effective?
* How long does the user usually need for assignments?
* What scheduling patterns improve completion?
* Which habits are maintained consistently?
* Which goals are frequently postponed?

This memory is shared across all AI agents.

---

# Objectives

The Memory System should:

* Learn continuously
* Preserve useful context
* Improve personalization
* Reduce repetitive questions
* Support long-term adaptation
* Respect user privacy
* Be explainable
* Scale efficiently

# Memory Types

The system uses three layers of memory.

---

# Short-Term Memory

Purpose:

Maintain context during a single request.

Examples:

* Current conversation
* Current task
* Current AI workflow
* Temporary variables

Lifetime:

* Seconds to minutes

Deleted automatically after workflow completion.

---

# Session Memory

Purpose:

Remember context throughout the user's active session.

Examples:

* Today's schedule
* Current focus session
* Temporary preferences
* Recently completed tasks
* Current productivity score

Lifetime:

* Hours

Automatically expires after inactivity.

---

# Long-Term Memory

Purpose:

Store persistent behavioral knowledge.

Examples:

* Preferred work hours
* Average task duration
* Reminder effectiveness
* Focus session preferences
* Habit streak history
* Goal completion patterns

Lifetime:

* Months to years
* User-controlled

---

# Memory Creation

New memories are created only when useful.

Examples:

Good memory:

> User consistently studies between 7 PM and 9 PM.

Bad memory:

> User opened the app at 5:02 PM.

Avoid storing insignificant events.

---

# Memory Retrieval

Before each AI workflow:

```text id="mem003"
Receive Request

↓

Identify Intent

↓

Retrieve Relevant Memory

↓

Inject Into Context

↓

Execute Agent
```

Agents never query the database directly.

---

# Context Ranking

Retrieved memories are ranked by:

* Relevance
* Recency
* Frequency
* Confidence
* Importance

Highest-ranked memories are injected into prompts.

---

# Vector Memory

Semantic memories are stored using embeddings.

Examples:

Task:

> Finish AI Assignment

Later:

> Complete machine learning homework

The vector database recognizes semantic similarity.

Applications:

* Similar task estimation
* Personalized recommendations
* Natural language search

---

# Embedding Strategy

Embed:

* Tasks
* Goals
* Reflections
* Coaching insights
* Habit notes
* User preferences

Do not embed sensitive credentials or secrets.

---

# Memory Confidence

Every memory stores a confidence score.

Example:

```json id="mem004"
{
  "memory": "User prefers coding after dinner.",
  "confidence": 0.92
}
```

Confidence increases when repeatedly confirmed.

---

# Memory Updates

Existing memories evolve.

Example:

Old:

> Focus session = 60 min

New observations:

90 min

90 min

90 min

Updated memory:

> Preferred focus session = 90 min

---

# Memory Decay

Outdated memories gradually lose importance.

Example:

Old preference:

Morning study

No longer observed.

↓

Confidence decreases.

↓

Eventually archived.

---

# Memory Pruning

Automatically remove:

* Outdated preferences
* Invalid assumptions
* Duplicate memories
* Low-confidence memories

Never delete explicit user preferences without confirmation.

---

# Storage Strategy

* PostgreSQL

Semantic memories:

* Vector Database (e.g., pgvector)

Temporary session data:

* Redis

This hybrid approach balances speed and intelligence.

# Security

The Memory System must:

* Encrypt stored memories.
* Respect user privacy settings.
* Isolate user memories.
* Prevent unauthorized access.
* Support complete memory deletion.
* Never store passwords, tokens, or sensitive credentials.

---

# Best Practices

* Store only meaningful memories.
* Continuously update confidence scores.
* Prefer recent, high-confidence memories.
* Separate structured and semantic memory.
* Explain AI behavior using stored memories.
* Allow users to inspect and manage stored knowledge.

---

# Antigravity Development Instructions

When implementing the AI Memory System:

1. Create a dedicated **Memory Service** accessed only through the AI Orchestrator.
2. Implement three memory layers: Short-Term, Session, and Long-Term.
3. Store structured memories in PostgreSQL and semantic memories using **pgvector**.
4. Build retrieval pipelines that rank memories by relevance, recency, confidence, and frequency.
5. Implement automatic confidence updates, decay, and pruning.
6. Provide APIs for creating, retrieving, updating, deleting, and exporting memories.
7. Inject only relevant memories into agent contexts to reduce prompt size.
8. Implement full user controls for viewing, deleting, resetting, and disabling AI memory.
9. Encrypt memory storage and isolate data per user.
10. Continuously monitor retrieval performance and learning effectiveness.
