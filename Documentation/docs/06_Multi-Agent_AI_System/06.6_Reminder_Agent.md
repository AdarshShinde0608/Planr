# Purpose

The **Reminder Agent** is responsible for ensuring users take action at the right moment.

Unlike traditional reminder systems that trigger notifications at fixed times, the Reminder Agent continuously evaluates:

* User availability
* Task urgency
* Calendar events
* Schedule changes
* User behavior
* Historical reminder effectiveness

It then decides:

* **Whether** to remind
* **When** to remind
* **How** to remind
* **How often** to remind
* **When to stop reminding**
* **When to escalate**

Its goal is **task completion**, not notification delivery.

---

# Context Awareness

Before sending any reminder, evaluate:

* Is the user currently busy?
* Is there an ongoing focus session?
* Is a meeting in progress?
* Is the task already completed?
* Has the reminder already been acknowledged?
* Are quiet hours active?

If any answer blocks the reminder, postpone it.

# Best Practices

* Prioritize action over notification volume.
* Send reminders only when they are likely to be effective.
* Adapt reminder timing using historical behavior.
* Respect user attention and quiet hours.
* Explain high-priority reminders.
* Stop reminders immediately after task completion.
* Coordinate with the Scheduler Agent before suggesting rescheduling.

---

# Antigravity Development Instructions

When implementing the Reminder Agent:

1. Build the Reminder Agent as an independent AI service coordinated by the AI Orchestrator.
2. Evaluate reminder necessity continuously using user context, schedule, and task status.
3. Support time-based, event-driven, and AI-driven reminder triggers.
4. Implement adaptive reminder frequency using historical effectiveness metrics.
5. Respect quiet hours, notification permissions, and user preferences.
6. Integrate with Firebase Cloud Messaging (FCM) for push notifications.
7. Coordinate with the Scheduler Agent for missed-task recovery and rescheduling.
8. Store reminder analytics to improve future reminder strategies.
9. Return all reminders using the defined JSON schema.
10. Ensure reminders remain supportive, non-intrusive, and focused on increasing task completion.
