# Purpose

The **Scheduler Agent** is responsible for transforming structured task plans into optimized daily schedules.

Unlike a traditional calendar, the Scheduler Agent continuously analyzes:

* User routine
* Fixed commitments
* Calendar events
* Planner Agent outputs
* User productivity patterns
* Task priorities
* Available free time

It then generates an adaptive schedule that maximizes task completion while minimizing stress and scheduling conflicts.

The Scheduler Agent is the primary differentiator of **The Last-Minute Life Saver**.

---

# Responsibilities

The Scheduler Agent is responsible for:

* Daily schedule generation
* Time-slot allocation
* Calendar synchronization
* Conflict detection
* Task sequencing
* Workload balancing
* Dynamic rescheduling
* Buffer insertion
* Break optimization
* Focus block generation
* Travel-time awareness
* Schedule explanation

# Time Slot Discovery

The Scheduler scans the day and identifies free windows.

Example:

```text
06:00–07:00

Available

08:00–17:00

Office

17:00–18:00

Travel

18:00–22:00

Available
```

Only available windows are considered for task placement.

---

# Slot Ranking Algorithm

Each free slot receives a score.

Factors:

* Energy level
* Focus score
* Deadline urgency
* Duration fit
* Calendar proximity
* User preference

Example:

| Slot        | Score |
| ----------- | ----: |
| 18:00–20:00 |    96 |
| 20:00–21:00 |    82 |
| 06:00–07:00 |    61 |

Highest-ranked slots are filled first.

---

# Task Allocation

Rules:

* Respect task dependencies
* Place difficult work first
* Group similar tasks
* Avoid excessive context switching
* Keep focus sessions uninterrupted

---

# Focus Blocks

Long tasks should be divided into focus sessions.

Example:

```text
Assignment

↓

90 Minutes

↓

15 Minute Break

↓

90 Minutes

↓

15 Minute Break

↓

60 Minutes
```

---

# Break Optimization

Breaks are automatically inserted.

Default:

| Focus Duration |  Break |
| -------------- | -----: |
| 60 min         | 10 min |
| 90 min         | 15 min |
| 120 min        | 20 min |

Breaks should not interrupt deep work unless necessary.

---

# Buffer Time

The Scheduler inserts buffer periods to absorb delays.

Examples:

* Transition between meetings
* Travel delays
* Unexpected interruptions

Default buffer:

* 10–15 minutes between major activities

---

# Conflict Detection

Detect:

* Overlapping tasks
* Calendar collisions
* Double bookings
* Overloaded days
* Insufficient available time

Example:

```text
Meeting

18:00–19:00

↓

Study Session

18:30–20:00

↓

Conflict Detected
```

---

# Conflict Resolution

Priority order:

1. Preserve fixed commitments
2. Protect high-priority tasks
3. Move flexible tasks
4. Split long tasks
5. Ask user if conflict remains

---

# Dynamic Rescheduling

Triggers:

* Task completed early
* Task delayed
* Calendar updated
* User skips session
* Reminder ignored

Workflow:

```text
Event

↓

Recalculate Availability

↓

Optimize Timeline

↓

Notify User
```

---

# Locking Schedule Blocks

Users may lock tasks.

Locked blocks:

* Cannot be moved automatically
* Participate in conflict analysis
* Influence future optimization

---

# Schedule Regeneration

Regeneration should preserve:

* Completed work
* Locked blocks
* Calendar events

Only remaining tasks are rescheduled.

---

# Optimization Objectives

The Scheduler optimizes for:

* Maximum completion rate
* Minimum deadline risk
* Balanced workload
* Reduced context switching
* Healthy breaks
* Peak productivity periods

# Best Practices

* Preserve user autonomy.
* Respect fixed commitments.
* Schedule difficult work during peak productivity.
* Prefer fewer, longer focus sessions over fragmented work.
* Insert recovery buffers.
* Explain scheduling decisions.
* Avoid unnecessary schedule changes.

---

# Antigravity Development Instructions

When implementing the Scheduler Agent:

1. Build the Scheduler Agent as an independent service invoked only through the AI Orchestrator.
2. Consume structured outputs from the Planner Agent rather than raw user prompts.
3. Implement the scheduling pipeline: availability detection → slot ranking → task allocation → break insertion → conflict resolution → optimization.
4. Support fixed, flexible, generated, and locked schedule blocks.
5. Integrate Google Calendar as a read/write provider with explicit user authorization.
6. Implement dynamic rescheduling triggered by task updates, calendar changes, and reminder events.
7. Return structured JSON containing schedules, focus blocks, conflicts, confidence, and explanations.
8. Persist generated schedules for analytics and future learning.
9. Validate every generated schedule against user constraints before returning it.
10. Never modify user schedules silently—major changes must require user confirmation.
