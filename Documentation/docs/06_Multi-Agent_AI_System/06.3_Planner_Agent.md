# Purpose

The **Planner Agent** is responsible for converting raw user input into structured, actionable work plans.

It is the first AI agent in the execution pipeline and serves as the foundation for all downstream agents.

The Planner Agent answers questions such as:

* What is the task?
* How difficult is it?
* How long will it take?
* When should it start?
* Should it be broken into subtasks?
* How urgent is it?
* What dependencies exist?
* What risks could prevent completion?

The output produced by the Planner Agent is consumed by:

* Scheduler Agent
* Coach Agent
* Reminder Agent

---

# Responsibilities

The Planner Agent is responsible for:

* Understanding user tasks
* Natural language interpretation
* Task classification
* Category prediction
* Priority estimation
* Time estimation
* Difficulty estimation
* Task decomposition
* Dependency detection
* Deadline risk analysis
* Recommendation generation

* Historical completion time

| Prepare Presentation |    5 hours |

# Recommendation Engine

Planner generates recommendations such as:

* Start today
* Break task into smaller sessions
* Reduce workload
* Delegate if possible
* Schedule during peak focus hours

Each recommendation includes a reason.

---

# Confidence Score

Every prediction includes a confidence value.

Example:

```json
{
  "priority": "High",
  "confidence": 0.91
}
```

Low-confidence predictions should be surfaced to the user for confirmation.

# Prompt Template

System Prompt:

```text
You are the Planner Agent for an AI productivity assistant.

Your responsibility is to analyze tasks and return structured JSON.

Do not generate schedules.

Do not generate reminders.

Focus only on task understanding, estimation, decomposition, and recommendations.
```

---

# LLM Output Contract

The Planner Agent must always return valid JSON.

Required fields:

```json
{
  "category": "",
  "priority": "",
  "difficulty": "",
  "estimatedDuration": 0,
  "recommendedStart": "",
  "deadlineRisk": "",
  "confidence": 0,
  "subtasks": [],
  "recommendations": []
}
```

Natural-language responses should never be returned directly to the orchestrator.

---

# Error Handling

If the task cannot be analyzed:

1. Return partial analysis.
2. Set confidence below 0.5.
3. Include clarification questions.
4. Never fabricate missing information.

# Best Practices

* Always return structured JSON.
* Keep recommendations explainable.
* Avoid assumptions without confidence scores.
* Use historical user data only when available.
* Keep subtasks independently executable.
* Prioritize transparency over complexity.
* Never schedule tasks directly.

---

# Antigravity Development Instructions

When implementing the Planner Agent:

1. Build the agent as an independent microservice or modular backend service.
2. Accept only structured requests from the AI Orchestrator.
3. Implement the complete planning pipeline: understanding → classification → estimation → decomposition → risk analysis.
4. Return responses using the defined JSON contract.
5. Include confidence scores and human-readable reasoning for all predictions.
6. Store planner outputs for future learning and accuracy evaluation.
7. Validate all LLM responses against a schema before returning them.
8. Gracefully handle incomplete or ambiguous tasks by requesting clarification.
9. Never perform scheduling, reminders, or coaching functions.
10. Expose the Planner Agent through a versioned internal API for future extensibility.
